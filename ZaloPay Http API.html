ZaloPay Http API

Published 2 years ago — 3 min read
By Nguyễn Thanh Thuận

Tổng quan

Trong qúa trình phát triển phần mềm thì các service có nhiều cách để giao tiếp với nhau, một trong những cách đó là Http API. Vậy làm sao để tạo được Http Server một cách nhanh hơn, config dễ dàng?
Với nỗ lực phát triển framework để mọi người dễ dàng sử dụng thì Vertx Http API được viết và đưa lên maven nội bộ của ZaloPay

Làm thế nào để build?

Trước khi bắt tay vào làm chúng ta sẽ pull dependencies trước nhé

<repository>
    <id>releases</id>
    <name>GP_ZaloPay-Storage-638.3-releases</name>
    <url>https://artifactory.zalopay.com.vn/artifactory/paycon</url>
</repository>

and add

<dependency>
    <groupId>vn.zalopay.vertx-http-api</groupId>
    <artifactId>zalopay-vertx-http-api</artifactId>
    <version>0.1.0</version>
</dependency>

Đến đây chúng ta sẽ tạo server vertx http với config đơn giản

Bước 1: Configuration thông qua ApiConfiguration

Có 2 cách là set port cho service như sau:

ApiConfiguration configuration = new ApiConfiguration(8080)

hoặc

ApiConfiguration configuration = ApiConfiguration.defaultConfiguration(8080)

Bước 2: Thêm service handler

Chúng ta có thể add một hoặc nhiều service handler tùy theo mục đích và nhu cầu riêng. Ở trong framework này đã được wrapped Json trong vertx handler vì thế không cần quan tâm nhiều về xử lý bytes to Json.

public static class Handler {
    public SimpleObject handleSuccess(RoutingContext rc, ApiJson requestObject) {
        //define type to record
        serviceMetrics.recordIngress(Constants.TYPE_SIMPLE);
        long startTime = System.currentTimeMillis();
        try {
            SimpleObject req = requestObject.getObject(SimpleObject.class);
            serviceMetrics.recordEgressSuccess(Constants.TYPE_SIMPLE, System.currentTimeMillis() - startTime);
            return req;
        } catch (Exception e) {
            System.err.println(ExceptionUtils.getStackTrace(e));
            serviceMetrics.recordEgressError(
                    Constants.TYPE_SIMPLE,
                    Constants.EXCEPTION,
                    System.currentTimeMillis() - startTime);
            return null;
        }
    }
}
    
//path `success`
configuration.registerRoute("success", HttpMethod.POST, handler::handleSuccess);

Bước 3: Start service

Http server

Cuối cùng chúng ta sẽ khởi vertx http tạo với configuration như trên

HttpApi httpApi = new HttpApi(configuration)

hoặc set thêm các vertxOption, deploymentOptions

HttpApi httpApi = new HttpApi(configuration, vertxOptions, deploymentOptions)

và start http server

httpApi.start()
Prometheus server

Trong framework này có tích hợp sẵn prometheus để monitor các metrics (mặc định: ingress, egress, latency, error) tùy vào nhu cầu có thể add thêm các metrics khác. Để start prometheus server cần truyền config, bên dưới là config ví dụ:

MetricsService metricsService =
        MetricsService.newBuilder()
                .setPort(9001)
                .setNamespace("zalopay")
                .setSubsystem("apiframework")
                .build();
metricsService.start();

như thế là chúng ta đã có Vertx Http server rồi đó. Bây giờ đến bước kế tiếp là gọi thử api success nhé. Mặc định trong framework này cũng có api handler ping pong /api/v1.0/ping

public static void main(String[] args) {
   OkHttpClient client = new OkHttpClient();
   String url = "http://localhost:8080/api/v1.0/success";
   Request request = new Request.Builder().url(url).build();

   try {
       Response response = client.newCall(request).execute();
       if (response.isSuccessful()) {
           System.out.println("Code=" + response.code());
           response.close();
           return;
       }

       System.err.println("Code=" + response.code() + " message=" + response.message());
   } catch (Exception ex) {
       System.err.println("Exception=" + ex.getMessage());
   }
}
